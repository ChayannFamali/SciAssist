# SciAssist Web

Веб-интерфейс для **SciAssist** — локального AI-ассистента для научной работы.

> Веб-обёртка над CLI-ядром. CLI-документация — в ветке [`master`](../../tree/master).

**Полностью offline** — никаких облаков, CDN и внешних API.  
Сервер слушает только `127.0.0.1`.

---

## Возможности

| | Возможность | Описание |
|---|---|---|
| 🕸 | **Граф статей** | Узел = статья. Сплошные рёбра — твои `[[ссылки]]`, пунктир — семантическая близость из ChromaDB |
| 📄 | **Чтение заметок** | Полный markdown любой статьи в браузере, `[[ссылки]]` кликабельны |
| ⊞ | **Сравнение** | 2–5 статей бок о бок по выбранной секции (Метод / Результаты / …) |
| ☰ | **Лента** | Те же статьи подряд, целиком |
| 💬 | **Ask** | RAG-вопрос с цитатами `[citekey]` — те же флаги, что в CLI |
| 🔍 | **Search** | Семантический поиск по чанкам без LLM |
| 📚 | **Library** | Каталог всех статей + статистика индекса |
| ⚙️ | **Process** | Запуск обработки статей с прогрессом по шагам |
| 📊 | **Logs** | Последние LLM-вызовы |

## Установка

Веб-интерфейс идёт как опциональная зависимость основного пакета.

```powershell
cd H:\SciAssist
.venv\Scripts\activate
pip install -e ".[web]"
```

Отдельная настройка не нужна — пути берутся из того же `.env`:  
`OBSIDIAN_VAULT`, `CHROMA_DB_PATH`, `LM_STUDIO_URL` и т.д.

### Предусловия

- Статьи уже обработаны: `sciassist process @citekey` хотя бы для нескольких штук — иначе граф и заметки пустые.
- LM Studio запущен с загруженными моделями — нужен для вкладок **Ask** и **Search**.
- Zotero запущен — нужен для вкладки **Library** (без него деградирует до read-only).

## Запуск

```powershell
sciassist serve
```

Открой в браузере: **http://127.0.0.1:8000**

## Как это устроено

Веб не «подключается к Obsidian» — Obsidian это просто читалка поверх папки `.md`.  
Веб читает **ту же папку** (`OBSIDIAN_VAULT` из `.env`) и строит граф сам.  
Obsidian и веб спокойно работают одновременно.

```
D:\SciVault\  (.md файлы с [[ссылками]])      ChromaDB (эмбеддинги)
      │                                              │
      └──────────────────┬───────────────────────────┘
                         ▼
                FastAPI (web/app.py)
           вызывает то же async-ядро, что и CLI:
             QueryEngine · RAGIndexer · ZoteroClient
                         │
                         ▼
                Фронтенд (web/static/)
         Cytoscape.js (граф) + markdown-it (заметки)
```

### Два типа связей в графе

```
сплошное ───  [[wiki-ссылки]]   — что ты связал осознанно
пунктир  ┄┄┄  семантика          — что близко по смыслу (top-K, с порогом)
```

Семантический слой показывает связи, которых нет в Obsidian: «эти две статьи ты не связал, но они про одно и то же». Клик по пунктиру → «Связать» — и ссылка вписывается в обе заметки.

## Типичный сценарий работы

```
1. sciassist process --queue      # обработать статьи в CLI (как обычно)
2. sciassist serve                # поднять веб
3. Браузер → вкладка Graph        # видишь библиотеку как граф
4. Режим "Оба"                    # сплошные связи + пунктир-подсказки
5. Клик по узлу                   # соседи подсвечены, справа — заметка
6. Ctrl-клик ещё 2–3 статьи       # мультивыбор
7. "⊞ Сравнить" → секция "Результаты"  # цифры всех статей рядом
8. Клик по пунктиру → "Связать"   # дописывает [[ссылку]] в обе заметки
9. Вкладка Ask                    # спросить по библиотеке с цитатами
```

## Вкладки

### Graph
Интерактивный граф библиотеки.

- **Режимы:** Связи / Смысл / Оба
- **Слайдер порога** — регулирует плотность семантических рёбер
- **Клик по узлу** — подсветка соседей + заметка в правой панели
- **Ctrl-клик / лассо** — мультивыбор 2–5 статей
- **Клик по пунктиру** — поповер с близостью и кнопкой «Связать»
- **⟳ Обновить** — пересчёт семантического слоя

### Панель заметок (справа)

**1 статья** — полный markdown, `[[ссылки]]` кликабельны.  
**2–5 статей** — переключатель:

- **⊞ Сравнить** — колонки + селектор секции + синхро-скролл
- **☰ Читать** — лента целиком, подряд

Мягкий лимит 5 статей (с подсказкой).

### Ask
RAG-вопрос с цитатами. Флаги: `top / hybrid / rerank / hyde / collection` — 1:1 с CLI.  
Ответ + таблица источников; «▶ к графу» подсвечивает статью в графе.

### Search
Семантический поиск — сырые чанки (citekey / section / score / preview), без LLM.

### Library
Карточки статистики индекса + список всех статей. Галочки → «показать выбранные» в панели заметок.

### Process
Запуск `process / analyze / gaps / draft` как фоновых задач с прогрессом по шагам.  
LM Studio обслуживает один запрос за раз — все генерации сериализуются.

### Logs
Последние LLM-вызовы из `data/logs/llm_calls.jsonl`.

## Что веб пишет в vault

Веб **только читает** vault и ChromaDB. Единственные исключения — точечные, атомарные правки:

| Действие | Что делает |
|---|---|
| «Связать» | Добавляет `[[ссылку]]` в секцию «🔗 Связи» обеих заметок |
| «Тег» | Добавляет тег во frontmatter |
| «Мысль» | Дописывает (append) в секцию «💡 Мои мысли» |

Запись атомарная, с проверкой `mtime` (защита от гонки с открытым Obsidian).  
Секции «💡 Мои мысли» и «📝 Заметки» **никогда не перезаписываются целиком**.

## Troubleshooting

| Симптом | Причина / решение |
|---|---|
| Граф пустой | Нет обработанных статей — запусти `sciassist process` |
| Нет пунктирных рёбер | Пустая `papers_notes` или порог слишком высокий — снизь слайдер |
| Ask / Search не отвечают | LM Studio не запущен или модель не загружена |
| Library пустая / ошибки | Zotero не запущен (работает read-only fallback) |
| `409 Conflict` при «Связать» | Файл изменён в Obsidian — обнови заметку и повтори |
| Долгая операция «висит» | Смотри `step` во вкладке Process; проверь, не занят ли LM Studio |
| Граф медленно строится | Семантический граф кэшируется; пересчёт — только по «⟳ обновить» |

## Известные ограничения

**Zotero write (501):** Local API не поддерживает запись — теги `processed` и перемещение в `SciAssist Processed` не работают. Статус отслеживается через `processed_registry.json`.

**Olmocr:** Требует anchor text — работает только если в PDF есть текстовый слой. Для чистых сканов нужна отдельная настройка.

**Qwen3 thinking mode:** Если включён — модель зависает. Отключи в LM Studio для всех используемых моделей.

**VLM контекст:** Qwen3-VL загружен с 4096 токенами. Крупные изображения автоматически ресайзятся до 1024px.

**Редактор заметок:** Полноценное редактирование тела заметки в браузере не реализовано — только точечные правки (link / tag / thought). Для полного редактирования используй Obsidian.
